# 安全说明

## 概述

本项目重视安全性，采取了多项措施来保护用户数据和通信安全。

## 安全特性

### 1. 密码安全

- **密码哈希**: 使用 Argon2id 算法进行密码哈希，这是当前最安全的密码哈希算法之一
- **密码强度要求**:
  - 长度必须在 8-20 字符之间
  - 必须包含至少一个大写字母
  - 必须包含至少一个小写字母
  - 必须包含至少一个数字

### 2. 认证安全

- **JWT 令牌**: 使用 JSON Web Token 进行无状态认证
- **Token 有效期**: Token 有效期为 24 小时
- **Token 刷新**: 支持 Token 刷新机制
- **Token 注销**: 支持 Token 黑名单机制，注销后立即失效
- **黑名单清理**: 过期 Token 自动从黑名单移除

### 3. 通信安全

- **TLS 支持**: 支持 TLS 加密传输
- **消息加密**: 所有敏感数据在传输过程中加密

### 4. 数据安全

- **SQL 注入防护**: 使用参数化查询和字符串转义
- **数据验证**: 所有输入数据都经过严格验证
- **最小权限原则**: 数据库操作遵循最小权限原则

### 5. 日志安全

- **日志分级**: 正常日志和错误日志分离
- **敏感信息保护**: 日志中不记录密码等敏感信息
- **日志轮转**: 防止日志文件过大，自动轮转

## 安全最佳实践

### 配置建议

1. **设置强 JWT 密钥**:
   - 建议至少 32 位随机字符
   - 使用密码管理器生成安全密钥
   - 定期更换密钥

2. **启用 TLS**:
   - 在生产环境中启用 TLS
   - 使用有效的 SSL/TLS 证书
   - 定期更新证书

3. **配置日志级别**:
   - 生产环境使用 `info` 级别
   - 开发环境可使用 `debug` 级别

### 部署建议

1. 使用防火墙限制访问
2. 定期更新依赖包
3. 监控异常日志
4. 定期备份数据库

## 漏洞报告

如果发现安全漏洞，请通过以下方式联系我们：

- 发送邮件至安全团队
- 创建安全相关的 Issue

我们承诺在收到报告后 48 小时内响应，并在合理时间内修复漏洞。

## 安全更新

我们会定期更新依赖包以修复安全漏洞。建议定期运行：

```bash
npm audit
npm update
```
